Test Plan for: EasyPoetry Application

Persona Type Tested: Primary, Secondary and Served

Context: At user home
Duration: 15 minutes

Complete the following before starting: Greeting and Consent & User Background Information.

Overall Objectives
  • Assess the overall effectiveness of the EasyPoetry application for different types of users performing basic, common tasks;
  • Identify obstacles in writing a poetry using the application;
  • Assess how easy is to find a good rhyme.
Scenario

  • The user want to write a poetry or song lyrics using the application.
  • The user want to customize the application interface.

Tasks
  • Open the application in a browser at the specified URL 20 seconds
  • Find a rhyme for a specified word - 20 seconds
  • Customize the application interface - 20 seconds
  • Writing a new poetry with 4 lyrics - 10 minutes
  • Search the definitions for unknown suggested words - 20 seconds
  • Select the poetry type - 10 seconds
  • Change the backround theme - 10 seconds
  • Change the backgound music - 10 seconds
Post-test Questions
  • What is you overall oppinion about the experience you had ?
  • What was the must difficult task for you ?
  • What do you think about the color scheme ?
Notes for Test Coordinator
  • Read the user the scenario for the test.
  • Read the user a task at the begining and once he/she finished the previus one read the next one.
  • Use a stopwatch.
  • Take pictures.

Test Results for: Product/Application Component

Success Criteria

A successful design has been achieved when:

  • 50% of users manged to complete the tasks in maximum 15 minutes

Summary

List major findings:

  • 1. Manually moving the rhyme from results list to poetry editor is not very intuitive.
  • 2. The lack of custom background, avatar and music.
  • 3. The text in the poetry editor is to small by default.
  • 4. The user want to have an easy way to find explanation for a set of sugested words without typing them again.
